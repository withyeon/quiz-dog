# 두근두근 인형뽑기 리뉴얼 계획서

## 목표

두근두근 인형뽑기를 단순 확률 보상 화면이 아니라, 퀴즈를 맞히는 행위가 자연스럽게 게임 실력과 수집 성과로 이어지는 완성도 높은 학습형 미니게임으로 리뉴얼한다.

## 현재 문제

- 정답을 맞히면 바로 뽑기 결과로 넘어가 조작감이 약하다.
- `onCorrectClick` 흐름이 비어 있어 퀴즈 피드백과 게임 보상이 분리되어 보인다.
- 인형뽑기 버튼이 실제 뽑기가 아니라 다음 문제 이동처럼 동작한다.
- 희귀도/점수는 주로 확률에 의존해 학습 성취와 연결되는 감각이 약하다.
- 인형뽑기 화면의 정보 구조가 수집, 순위, 학습 보상을 선명하게 보여주지 못한다.

## 리뉴얼 콘셉트

정답을 맞히면 크레인 전력이 충전되고, 학생은 인형뽑기 기계에서 직접 타이밍을 맞춰 집게를 내린다. 빠르게 맞힌 정답, 연속 정답, 정확한 조작은 점수와 희귀도에 반영된다. 오답은 기회를 놓치지만 바로 다음 문제로 복귀해 학습 흐름이 끊기지 않도록 한다.

## 핵심 게임 루프

1. 퀴즈를 푼다.
2. 정답이면 크레인 전력 1회와 속도 보너스를 얻는다.
3. 정답 피드백을 확인한 뒤 인형뽑기 화면으로 이동한다.
4. 움직이는 집게를 타이밍에 맞춰 내린다.
5. 정답 속도, 기계 등급, 조작 정확도, 아이템 효과로 인형과 점수가 결정된다.
6. 결과 카드를 확인하고 다음 퀴즈로 돌아간다.

## 학습 연결 설계

- 빠른 정답: 인형 점수 보너스가 커진다.
- 연속/누적 정답: 기계 등급이 올라가 희귀 인형 확률이 상승한다.
- 정답 후 직접 조작: 퀴즈 보상이 즉시 게임 행동으로 연결된다.
- 결과 화면: 이번 정답의 속도와 조작 정확도를 함께 보여준다.
- 컬렉션/순위: 반복 학습이 수집량과 점수 경쟁으로 드러난다.

## 구현 범위

- `lib/game/fishing.ts`
  - 인형뽑기 보상 계산을 정답 속도, 기계 등급, 조작 정확도 기반으로 개편한다.
  - 속도/조작 등급과 기계 등급 진행률 유틸을 추가한다.

- `hooks/useFishingGame.ts`
  - 정답 즉시 뽑기에서 정답 후 대기 보상 흐름으로 변경한다.
  - 크레인 조준 상태, 목표 위치, 조작 정확도, 결과 시퀀스를 관리한다.
  - 아이템은 다음 뽑기 보정으로 동작하게 정리한다.

- `components/FishingMachine.tsx`
  - 움직이는 집게, 목표 존, 드롭 버튼, 수집 인형 레이어를 갖춘 조작형 기계로 재구성한다.
  - 게임 상태별 피드백과 보너스 정보를 시각적으로 보여준다.

- `app/fishing/page.tsx`
  - 퀴즈 화면에 학습 보상 패널, 컬렉션, 순위를 배치한다.
  - 정답 피드백 클릭 시 실제 인형뽑기 조작 화면으로 이동하게 연결한다.
  - 결과 카드에 점수, 속도, 조작 정확도를 함께 표시한다.

## 완료 기준

- 퀴즈 정답 후 자동 결과가 아니라 조작형 인형뽑기 화면으로 이어진다.
- 정답 속도와 조작 정확도가 결과에 반영된다.
- 기존 `fishing` 모드, DB 컬럼(`caught_dolls`, `claw_points`, `score`)과 호환된다.
- 빌드가 통과하고 학생 플레이 화면에서 주요 상태 전환이 깨지지 않는다.
